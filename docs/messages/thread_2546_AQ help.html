From: thedoubtfulguests | Date: 2007-07-04 17:59:28 UTC
I have wrestled with this problem a lot with the Circus event
stamps. The question is whether the stamps are available "on the
table", or are they Planted or hidden? Do they require enough work
to justify counting them as "Finds" for the purpose of PFX counts?

Planted event stamps -- Traditional or Mystery - Any stamp at an
event that can be counted as a find I think should be separately
listed as a Traditional box or a Mystery box. That way when people
Record them they count in PFX.

A Traditional listing will show up with the gather by location as a
nearby stamp. If it is at the gather for only one day this is kind
of a waste. Instead you can list the stamp as a Mystery box with no
location. Then the same listing can be used over and over. The only
problem is there is no easy way to associate the box listing with
the event. I like the idea of using the event name in the box name,
like "Great Castle Quest: Urquart Castle". If it attends another
event, I dare you to just rename the box with the next event,
like "Scottish Fling: Urquart Castle". Then you can just list in the
clues all of the events it went to.

For 'Great Castle Quest', the hosts listed all the planted stamps as
a Mystery box with multiple stamps. Then they just wrote all
attendees where they could find them for recording. Not hard since a
Basic Search for "Great Castle Quest" will find them.

For 'Summer Slam Pub Crawl' he listed all the boxes as an Event box
with multiple stamps and he could then tie the listing to the event.

On the Table Event stamps -- These are the traveling event stamps
for which the "Event" stamp listing was created. Stamps like 'K-
MartHa", "LbNA Map", and "Boch's Circus". They get one listing and
can be assigned to one event after another. This makes it easy to
keep track of where they are and where they've been.

Only problem is that they do not count as "Finds". For "on the
table" this is appropriate. But I encourage people to plant Boch's
Circus so this makes it complicated. Most people just use the Event
style listing anyway and not worry about the count.

Oh I do try to complicate things to manipulate the system don't I?
We sure make it difficult for Ryan with all of out permutations.
