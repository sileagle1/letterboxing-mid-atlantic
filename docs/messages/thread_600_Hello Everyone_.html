It has also been recommended that you do what realtors do.
After a series of attacks where realtors were lured to empty homes on the premise of
showing them, it is now policy by most realtors to leave information about where and
when you are going with a contact person or on a voice mail system. If plans change, you
first call the contact or voice mail before you go. This is, in fact how the attacker was
caught.

So if you are at all concerned, call a friend or your own voice mail and leave the name of
the area you are going to, the name of the box you are hunting, and the time you expect
to be back. If you find that the hunt takes you away from that area or it will take longer
than you expect (we've all been there racing sunset) call and leave an update - including
the time of the update.

This is a good policy for everyone, as a broken leg in the winter woods can be as deadly as
an attacker.

Re: One word. Mac-friendly

From: hexacat | Date: 2004-12-01 17:42:36 UTC
ok. i hear the cries from all Mac users and i wish i could help;
honest. i don't own a Mac so i can only offer suggestions...

A few choices, none very good.

Best choice: Use Timbuktu, to connect to a PC running Access. Install
a client on the Mac, install a client on the PC, configure, and you
can connect to it anytime you like and do whatever as if you were
sitting at the PC, short of physically touching it.

Worst Choice: Install VPC, install Access, constantly be frustrated
with slow performance and crashing.


Second best choice (well, maybe my first): publish the Access DB
using a web based front end. It has builtin wizards for doing that.

The real issue is that I don't think you can read a .mdb file with
any ODBC driver available on the Mac. That, and that Redmond keeps
changing the damn file format with each release.

Which leads to the third choice (but a potentially really expensive
one): upscale the databases to SQL Server, where there are ODBC
drivers on the Mac. Then you can create a front end in FileMaker or
4D and deploy it crossplatform and lose Access altogether. Be still
my heart! I like you guys but the buck $$ stops here.


If you are keeping a PC with Access on it there is:

http://www.microsoft.com/mac/DOWNLOAD/MISC/RDC.asp


Remote Desktop Connection Client 1.0.1 for Mac

Remote Desktop Connection Client 1.0.1 for Mac allows you to connect
to a Windows-based computer and work with programs and files on that
computer from your Macintosh computer. To connect to a Windows-based
computer, you need network access and permissions to connect to the
computer, and the computer must be running Terminal Services or
Remote Desktop Services. The following Windows products support
Remote Desktop connections:
Windows XP Professional
Windows Server 2003
Windows 2000 Server
Windows 2000 Advanced Server
Windows 2000 Datacenter Server
Windows NT Server 4.0, Terminal Server Edition
